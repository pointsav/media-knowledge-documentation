### Panel de administración / Administrador del sistema / Panel de control / Preferencias del sistema
En PointSav, la administración del sistema se realiza a través del libro de comandos os-console y las interfaces web autenticadas por MBA; no existe un panel de administración centralizado — toda acción privilegiada es una transacción firmada y auditable.

### Inteligencia artificial (IA)
*Artificial Intelligence or AI*

Teoría y desarrollo de sistemas informáticos capaces de realizar tareas que normalmente requieren inteligencia humana, como la percepción visual, el reconocimiento de voz, la toma de decisiones y la traducción entre idiomas.

Los datos de audiencia describen las características y preferencias de los destinatarios de una comunicación; en PointSav, se derivan de los registros de identidad de service-people y las puntuaciones de gravedad de service-content, sin recurrir a intermediarios de datos externos.

## D
<hr>

### Entrada de datos
La entrada de datos es el ingreso manual de información estructurada en un sistema; en PointSav, todas las escrituras de componentes de IA están prohibidas — solo las escrituras deterministas verificadas por humanos alcanzan el grafo de conocimiento, conforme a SYS-ADR-07.

### Migración de datos
La migración de datos es la transferencia de datos entre sistemas de almacenamiento o formatos; en PointSav, las migraciones las gestiona el pipeline de ingesta determinista de service-content y están sujetas a la garantía de solo adición del worm-ledger-design.

### Red descentralizada
Una red descentralizada distribuye funciones y datos en múltiples nodos sin una autoridad de control única; PointSav lo implementa como una topología de malla soberana con enrutamiento determinista y anclaje criptográfico en os-infrastructure.

### Sistema de diseño
Un sistema de diseño es una colección de componentes reutilizables, tokens y patrones documentados que estandarizan el lenguaje visual e interactivo de un producto; la implementación de PointSav es pointsav-design-system, una bóveda en formato DTCG entregada como repositorio Git.

### Componentes del sistema de diseño
Los componentes del sistema de diseño son los bloques de construcción UI reutilizables individuales —botones, barras de navegación, campos de formulario— documentados en pointsav-design-system/components/ con recetas HTML+CSS+ARIA y referencias de tokens.

- **Permisos basados en dispositivo**

### DevOps
*DevOps*

Conjunto de prácticas que combina el desarrollo de software (Dev) y las operaciones de TI (Ops). Su objetivo es acortar el ciclo de vida del desarrollo de sistemas y proporcionar entrega continua con alta calidad de software. DevOps es complementario al desarrollo ágil de software; varios aspectos de DevOps provienen de la metodología ágil.

### Transformación digital
*Digital Transformation*

Proceso de usar tecnologías digitales para crear nuevos procesos de negocio —o modificar los existentes— cultura y experiencias de clientes para responder a los requisitos cambiantes del negocio y el mercado.

### Gemelo digital
*Digital Twin*

Modelo virtual diseñado para reflejar con precisión un objeto físico. En el contexto de PointSav, se refiere a una réplica digital de un edificio u otro activo físico.

- **Aviso legal**

### Red distribuida
Una red distribuida distribuye el cómputo y el almacenamiento entre muchos nodos, cada uno capaz de operar de forma independiente; la flota de PointSav lo implementa a través de os-orchestration coordinando múltiples archivos ToteboxOS.

### Docker
*Docker*

Conjunto de productos de plataforma como servicio (PaaS) que usan virtualización a nivel de SO para entregar software en paquetes llamados contenedores. Los contenedores están aislados entre sí y agrupan su propio software, bibliotecas y archivos de configuración; pueden comunicarse entre sí a través de canales bien definidos.

- **Documentación**
- **Flujo descendente (downstream)**

### service-extraction
*service-extraction*

El servicio analizador determinista. Elimina el formato propietario de los datos entrantes y enruta los datos estructurados a servicios deterministas y el texto no estructurado a service-slm. Asigna identificadores de transacción para el rastreo de cadena de custodia. Nombre canónico de largo plazo que reemplaza el nombre de trabajo heredado service-parser.

### service-search
*service-search*

El servicio de índice invertido. Proporciona recuperación rápida de texto completo en todos los archivos de un archivo Totebox usando Tantivy. Opera sin un motor de base de datos en ejecución — el índice es estático y binario, y puede consultarse en una máquina sin conexión. Cumple la conformidad DARP: los datos deben ser consultables sin software propietario.

### SO de Workplace
*Workplace OS*

Sistema operativo gráfico y fácil de usar, diseñado para tareas empresariales cotidianas, que proporciona un conjunto de aplicaciones de productividad y un entorno seguro para los usuarios.

### SO de Workplace — Navegador
*Workplace OS – Browser*

Navegador web integrado en SO de Workplace que proporciona acceso seguro y controlado a internet y a las aplicaciones web internas.

### SO de Workplace — Calculadora
*Workplace OS – Calculator*

Aplicación básica de calculadora incluida en SO de Workplace para realizar cálculos matemáticos sencillos.

### SO de Workplace — Gestor de archivos
*Workplace OS – File Manager*

Aplicación de gestión de archivos en SO de Workplace para organizar, navegar y administrar archivos y carpetas.

### SO de Workplace — Visor de PDF
*Workplace OS – PDF Viewer*

Aplicación para ver documentos PDF dentro del entorno SO de Workplace.

### SO de Workplace — Hoja de cálculo
*Workplace OS – Spreadsheet*

Aplicación de hojas de cálculo en SO de Workplace para crear y editar hojas de cálculo, realizar cálculos y analizar datos.

### SO de Workplace — Procesador de textos
*Workplace OS – Word Processor*

Aplicación de procesamiento de textos en SO de Workplace para crear, editar y dar formato a documentos de texto.
